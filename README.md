# Forecast Variance Knowledge Graph

A ServiceNow custom application that adds a Natural Language Query (NLQ) interface and Neo4j-style graph visualization on top of the native Knowledge Graph engine.

## What it does

- Calls the native ServiceNow NLQ processor (`NLQVisualization`) through a Scripted REST API
- Parses the generated `filter_query` and `field_list` to run a focused `GlideRecord` query
- Shows the matching records as a dynamic table
- Renders the first 5 sample records as an interactive force-directed graph
- Keeps the raw native NLQ output available behind a toggle

## Prerequisites

- Node.js 18+
- ServiceNow instance with the `x_snc_forecast_v_0` forecast variance app installed
- ServiceNow user with admin or scoped-application rights
- (Optional) Python 3.10+ for table-creation helpers

## Setup

1. Copy the environment template:

   ```bash
   cp .env.example .env
   ```

2. Fill in `.env` with your instance, credentials, and the target app sys_id.

3. Install dependencies:

   ```bash
   npm install
   ```

4. Add SDK authentication:

   ```bash
   python3 scripts/add_sdk_auth.py
   ```

5. Build and deploy:

   ```bash
   npx @servicenow/sdk build
   npx @servicenow/sdk deploy --auth forecast-kg
   ```

6. Create the bridge table (if it does not exist):

   ```bash
   pip install requests python-dotenv
   python3 scripts/create_cost_center_bridge_table.py
   ```

7. Open the portal:

   ```
   https://<instance>.service-now.com/forecast-kg-portal?id=forecast_kg_page
   ```

## Notes

- `src/fluent/generated/keys.ts` is instance-specific and is regenerated by the SDK on deploy.
- The `unused/` folder contains deprecated Snowflake ZCC work that is not part of the final setup.
